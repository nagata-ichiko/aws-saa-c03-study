# Q4: 不正解記録 (20260325)

**ドメイン:** D2

## 問題

あなたのウェブサイトでは、従業員向けにオンデマンドのトレーニング動画を配信しています。動画は毎月高解像度のMP4形式でアップロードされます。従業員は世界中に分散しており、移動中に会社支給のタブレットを使って動画を視聴します。このタブレットはHTTP Live Streaming（HLS）プロトコルに対応している必要があります。会社には動画のトランスコーディングの専門知識がなく、場合によってはコンサルタントに支払う必要があるかもしれません。高い可用性と動画配信の品質を損なわずに、最もコスト効率の良いアーキテクチャをどのように実装しますか？

## 選択肢

- **A.** SQSでタスクを分散し、キューの長さに応じてAuto Scalingでノード数を調整するEC2上の動画トランスコーディングパイプライン。動画はEBSボリュームに保存し、数日後にEBSスナップショットで元ファイルを増分バックアップ。CloudFrontでEC2からHLSトランスコード済み動画を配信。
- **B.** Elastic Transcoderで高解像度MP4動画をHLSにトランスコード。動画はEBSボリュームに保存し、数日後にEBSスナップショットで元ファイルを増分バックアップ。CloudFrontでEC2からHLSトランスコード済み動画を配信。
- **C.** Elastic Transcoderで高解像度MP4動画をHLSにトランスコード。動画はS3に保存し、ライフサイクル管理で数日後に元ファイルをGlacierにアーカイブ。CloudFrontでS3からHLSトランスコード済み動画を配信。
- **D.** SQSでタスクを分散し、キューの長さに応じてAuto Scalingでノード数を調整するEC2上の動画トランスコーディングパイプライン。動画はS3に保存し、ライフサイクル管理で数日後にすべてのファイルをGlacierにアーカイブ。CloudFrontでGlacierからHLSトランスコード済み動画を配信。

## あなたの回答: 
## 正解: C
