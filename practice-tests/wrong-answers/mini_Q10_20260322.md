# 間違えた問題: DynamoDB 読み取りキャッシュ

- **日付**: 2026-03-22
- **モード**: ミニ模擬試験
- **問題番号**: Q10（セッション内）

## 問題文

ある企業が、Amazon DynamoDB テーブルに大量のデータを保存しています。特定のアイテムに対して頻繁に読み取りリクエストを行いますが、データの更新頻度は低いです。読み取りパフォーマンスを向上させ、DynamoDB テーブルへの読み取りリクエスト数を削減するために最も適切なソリューションは？

## 選択肢

- A. DynamoDB のプロビジョンドキャパシティモードで RCU を増加
- B. DynamoDB Accelerator（DAX）を使用してキャッシュ
- C. DynamoDB グローバルテーブルで複数リージョンにレプリケート
- D. Amazon ElastiCache for Redis を DynamoDB の前に配置

## 自分の回答: D（ElastiCache for Redis）

## 正解: B（DAX）

## 解説

- DAX は DynamoDB 専用のインメモリキャッシュ（マイクロ秒レベルのレイテンシー）
- DynamoDB API と互換のため、アプリケーションコードの変更がほぼ不要
- ElastiCache でも可能だが、キャッシュロジックを自前実装する必要があり運用負荷が高い
- SAA 試験: DynamoDB のキャッシュ = DAX が鉄則
