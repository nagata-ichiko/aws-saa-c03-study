# Q520 - サブネットと AZ の関係

- **日付**: 2026-03-19
- **分野**: ネットワーク / VPC
- **自分の回答**: B（正しい）
- **正解**: C（誤りです）

## 問題文

VPC は複数のサブネットを含みますが、それぞれのサブネットは複数のアベイラビリティゾーン（AZ）にまたがることができますか？

## 選択肢

- A. これは VPC のセットアップ時に要求した場合のみ当てはまります。
- B. これは正しいです。
- **C. これは誤りです。** ← 正解
- D. これは米国リージョンのみ当てはまります。

## 解説

**サブネットは必ず1つの AZ にのみ属します。複数の AZ にまたがることはできません。**

| 概念 | AZ との関係 |
| :--- | :--- |
| VPC | リージョン全体にまたがる（複数 AZ を含む） |
| **サブネット** | **1つの AZ にのみ属する（またがれない）** |
| EC2 インスタンス | 1つのサブネット（= 1つの AZ）に配置される |

高可用性を実現したい場合は、複数の AZ にそれぞれサブネットを作成して、各 AZ にリソースを分散させる。

## 覚え方

「1サブネット = 1 AZ」。SAA-C03 の頻出ポイント。
