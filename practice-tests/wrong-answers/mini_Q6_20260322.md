# 間違えた問題: 地理的アクセス制限 + SQL インジェクション対策

- **日付**: 2026-03-22
- **モード**: ミニ模擬試験
- **問題番号**: Q6（セッション内）

## 問題文

ある企業が、Amazon EC2 インスタンス上で動作する Web アプリケーションを運用しています。アプリケーションは Application Load Balancer (ALB) の背後に配置されています。企業は、特定の国からのアクセスをブロックし、SQL インジェクション攻撃からアプリケーションを保護したいと考えています。

## 選択肢

- A. ALB のセキュリティグループを使用して、特定の国の IP アドレス範囲をブロックし、アプリケーションコードで SQL インジェクション対策を実装する
- B. Amazon CloudFront を ALB の前に配置し、CloudFront の地理的制限機能で特定の国をブロックし、AWS WAF で SQL インジェクション対策のルールを適用する
- C. AWS Shield Advanced を有効にして、特定の国からのトラフィックと SQL インジェクション攻撃の両方をブロックする
- D. Network ACL を使用して特定の国の IP アドレスをブロックし、Amazon Inspector で SQL インジェクションの脆弱性をスキャンする

## 自分の回答: C（AWS Shield Advanced）

## 正解: B（CloudFront + WAF）

## 解説

- AWS Shield Advanced は DDoS 防御が主機能。地理的制限や SQL インジェクション対策は不可
- CloudFront の Geo Restriction で国単位のブロックが可能
- AWS WAF の SQL インジェクションルールでアプリケーション層を保護
