# 間違えた問題: Q399

- **日付**: 2026-03-23
- **ドメイン**: Domain 3: 高パフォーマンスアーキテクチャ
- **自分の回答**: D (Hadoop Web Interfaces)
- **正解**: A (Kinesis)

## 問題

ゲームサイトのために大量のデータを分析するのにEMRが有効だと判断し、Amazon EMRクラスターをデプロイしました。クラスターのパフォーマンスを監視したい場合、次のうち**クラスターのパフォーマンス監視に使用できないツール**はどれですか？

- A. Kinesis
- B. Ganglia
- C. CloudWatch Metrics
- D. Hadoop Web Interfaces

## 解説

**正解の理由**: Kinesisはリアルタイムのストリーミングデータ処理サービスであり、EMRクラスターのパフォーマンス監視用ツールではありません。監視向けのメトリクス取得や可視化には適していません。

**誤答のポイント**: Gangliaはクラスターモニタリングツール、CloudWatch MetricsはAWSの監視サービス、Hadoop Web Interfacesはジョブやノードの状態確認に使われるため、いずれもパフォーマンス監視に有効です。

**関連知識**: AWS試験では、EMRの監視方法（CloudWatch、Ganglia、Hadoop UI）や用途ごとのサービスの役割を理解する問題が頻出です。Kinesisはデータストリーミング用途と明確に区別しましょう。
