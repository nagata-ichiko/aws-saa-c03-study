# Q688 - EC2 コスト分析ツールの選択

- **日付**: 2026-03-19
- **分野**: コスト最適化
- **自分の回答**: D（Cost and Usage Reports + QuickSight）
- **正解**: B（Cost Explorer）

## 問題文

ある企業が最新の請求書で Amazon EC2 のコスト増加を確認しました。請求チームは、数台の EC2 インスタンスで意図しない垂直スケーリングが発生していることに気づきました。ソリューションアーキテクトは、過去2か月の EC2 コストを比較するグラフを作成し、垂直スケーリングの根本原因を特定するために詳細な分析を行う必要があります。**最も運用負荷が少ない**方法はどれですか？

## 選択肢

- A. AWS Budgets を使って予算レポートを作成し、インスタンスタイプ別の EC2 コストを比較する。
- **B. Cost Explorer の詳細なフィルタリング機能を使って、インスタンスタイプ別の EC2 コストを詳細に分析する。** ← 正解
- C. AWS Billing and Cost Management ダッシュボードのグラフを使って、過去2か月のインスタンスタイプ別 EC2 コストを比較する。
- D. AWS Cost and Usage Reports を使ってレポートを作成し、Amazon S3 バケットに送信する。Amazon QuickSight を S3 をデータソースとして使い、インスタンスタイプ別のインタラクティブなグラフを作成する。

## 解説

| 選択肢 | ツール | 運用負荷 | 問題点 |
| :--- | :--- | :--- | :--- |
| A | AWS Budgets | 低 | 予算管理用。コスト分析・グラフ作成には不向き |
| **B** | **Cost Explorer** | **最低** | インスタンスタイプ別フィルタリング・グラフ表示が標準機能として内蔵。追加設定不要 |
| C | Billing ダッシュボード | 低 | 概要表示のみ。インスタンスタイプ別の詳細分析は不可 |
| D | Cost and Usage Reports + S3 + QuickSight | 高 | 強力だが、S3 バケット設定・QuickSight セットアップなど多くの追加作業が必要 |

## 覚え方

- 「コスト分析をすぐやりたい・運用負荷最小」→ **Cost Explorer**
- 「大規模なカスタム分析・BI ツールと連携」→ Cost and Usage Reports + QuickSight
