AI エージェント（Claude Code / Manus）に指示して、新しいカードの追加や問題形式での出題を行わせてください。

## 🔐 セキュリティ・IAM

**Q: IAM ポリシーの評価ロジックで、明示的な Deny と Allow が競合した場合はどうなるか？**
A: 明示的な Deny が常に優先される。Allow があっても Deny があれば拒否される。

**Q: AWS Organizations の SCP (Service Control Policy) の役割は？**
A: メンバーアカウントで使用できる AWS サービスとアクションを制限するポリシー。SCP は権限を付与しない（許可の上限を設定するだけ）。

**Q: IAM ロールと IAM ユーザーの主な違いは？**
A: IAM ユーザーは長期的な認証情報（パスワード・アクセスキー）を持つ。IAM ロールは一時的な認証情報を使用し、AWS サービスや外部 ID プロバイダーが引き受けることができる。

## 🌐 ネットワーク・VPC

**Q: VPC エンドポイントの2種類とその違いは？**
A: ① ゲートウェイ型（S3・DynamoDB のみ、ルートテーブルに追加）② インターフェイス型（PrivateLink を使用、ENI を作成、多くのサービスに対応）

**Q: NAT ゲートウェイと NAT インスタンスの主な違いは？**
A: NAT ゲートウェイはマネージドサービスで高可用性・高スループット。NAT インスタンスは EC2 インスタンスで管理が必要だが、セキュリティグループを設定でき、コストが低い。

**Q: VPC ピアリングの制限事項は？**
A: トランジティブルーティング不可（A-B-C の場合、A から C には直接通信できない）、CIDR の重複不可、リージョンをまたぐことは可能。

## 💾 ストレージ

**Q: S3 ストレージクラスを安価な順に並べると？**
A: S3 Glacier Deep Archive < S3 Glacier Flexible Retrieval < S3 Glacier Instant Retrieval < S3 One Zone-IA < S3 Standard-IA < S3 Intelligent-Tiering < S3 Standard

**Q: EBS ボリュームタイプ gp2 と gp3 の違いは？**
A: gp3 は gp2 より 20% 安価で、IOPS とスループットを独立して設定可能（gp2 はボリュームサイズに依存）。gp3 の基本 IOPS は 3000（gp2 は 100）。

**Q: EFS と FSx の違いは？**
A: EFS は Linux 向けの NFS ファイルシステム（マネージド）。FSx は Windows 向け（FSx for Windows File Server）や高性能 HPC 向け（FSx for Lustre）など複数の選択肢がある。

## ⚡ コンピューティング

**Q: EC2 購入オプション（オンデマンド・リザーブド・スポット・Dedicated）の使い分けは？**
A: オンデマンド：短期・不規則なワークロード。リザーブド：予測可能な長期ワークロード（最大72%割引）。スポット：中断可能なバッチ処理（最大90%割引）。Dedicated：コンプライアンス要件がある場合。

**Q: Auto Scaling の3種類のスケーリングポリシーは？**
A: ① ターゲット追跡スケーリング（メトリクスを目標値に維持）② ステップスケーリング（アラームに応じて段階的に調整）③ シンプルスケーリング（アラームに応じて固定量を調整）

## 🗄️ データベース

**Q: RDS Multi-AZ と Read Replica の違いは？**
A: Multi-AZ は可用性向上のためのスタンバイ（フェイルオーバー用、読み取り不可）。Read Replica は読み取りパフォーマンス向上のためのレプリカ（読み取り可能、非同期レプリケーション）。

**Q: DynamoDB の RCU (Read Capacity Unit) の計算方法は？**
A: 強整合性読み取り：4KB/RCU。結果整合性読み取り：8KB/RCU（0.5 RCU）。トランザクション読み取り：4KB/2 RCU。
