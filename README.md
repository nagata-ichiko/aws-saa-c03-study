# AWS Certified Solutions Architect - Associate (SAA-C03) 学習リポジトリ

このリポジトリは、AWS Certified Solutions Architect - Associate (SAA-C03) 資格取得に向けた学習を管理するためのワークスペースです。
Claude Code や Manus などの AI エージェントと連携し、効率的に学習を進めることを目的としています。

## 🎯 学習目標

- **資格**: AWS Certified Solutions Architect - Associate (SAA-C03)
- **目的**: AWS の主要サービスに関する深い理解と、ベストプラクティスに基づいたアーキテクチャ設計スキルの習得

## 📂 ディレクトリ構造

```text
.
├── notes/                  # 学習ノート
│   ├── domain1-secure-architecture/    # 第1分野: セキュアなアーキテクチャの設計 (30%)
│   ├── domain2-resilient-architecture/ # 第2分野: 弾力性に優れたアーキテクチャの設計 (26%)
│   ├── domain3-high-performance/       # 第3分野: 高性能アーキテクチャの設計 (24%)
│   ├── domain4-cost-optimized/         # 第4分野: コストを最適化したアーキテクチャの設計 (20%)
│   └── services/                       # サービス別の詳細ノート
├── practice-tests/         # 模擬テスト・問題演習
│   ├── mock-exams/         # 模擬試験の記録
│   ├── wrong-answers/      # 間違えた問題の管理と復習ノート
│   └── flashcards/         # 暗記用フラッシュカードデータ
├── progress/               # 学習進捗管理
│   └── study-plan.md       # 学習計画と進捗トラッキング
├── resources/              # 参考資料リンク集
│   └── references.md       # 公式ドキュメントやホワイトペーパーのリンク
└── .github/                # GitHub 関連設定
    └── ISSUE_TEMPLATE/     # 復習用 Issue テンプレート
```

## 🤖 AI エージェント（Claude Code / Manus）との連携方法

このリポジトリは、AI エージェントを活用して学習効率を最大化するように設計されています。

### 活用例

1. **概念の解説とノート作成**
   - 「AWS KMS と Secrets Manager の違いを `notes/services/security.md` にまとめて」
   - 「Domain 1 のセキュアなアーキテクチャ設計について、重要なポイントを抽出して」

2. **模擬問題の生成と採点**
   - 「VPC エンドポイントに関する SAA レベルの模擬問題を3問作成して」
   - 「この解答が間違っている理由を解説し、`practice-tests/wrong-answers/` に記録して」

3. **学習進捗の管理**
   - 「今日の学習内容を `progress/study-plan.md` に追記して」
   - 「試験日まであと1ヶ月です。残りの学習計画を最適化して」
